## 1. 프로젝트 개요

### 1-1. 비즈니스 배경

한국 디지털 광고 시장에서 YouTube는 핵심 채널로 자리잡았으나, 광고주는 어떤 크리에이터/콘텐츠에 광고를 집행할지 결정할 객관적 기준이 부재한 상황입니다. 트렌딩 영상의 카테고리별 반응 패턴과 지속 시간에 대한 데이터 기반 분석이 없어 광고 집행이 주로 경험과 직관에 의존합니다.

### 1-2. 비즈니스 목표

YouTube KR 트렌딩 데이터(2022~2025)를 분석하여 카테고리별 트렌딩 지속 패턴을 발견하고, 초기 반응 신호만으로 트렌딩 지속 시간을 예측하는 ML/DL 모델을 구현합니다.  
이를 통해 **한국 광고주가 크리에이터 광고 집행 의사결정을 데이터 기반으로 수행**할 수 있도록 지원합니다.

### 1-3. 머신러닝 활용 계획

| 항목 | 내용 |
|------|------|
| 문제 유형 | 회귀 (trending_duration_h 예측) + 이진 분류 (tdi_label 예측) |
| 예측 대상 | 트렌딩 지속 시간(h) 및 트렌딩 지속성 등급(0/1) |
| 주요 모델 | XGBoost (회귀/분류), MLP (회귀/분류) |
| 해석 방법 | SHAP을 활용한 피처 중요도 분석 |
| 활용 방안 | 광고주가 크리에이터 선정 시 트렌딩 지속성 예측값 참고 |
| 기대 효과 | 광고 집행 효율 향상 및 카테고리별 최적 집행 시점 제안 |

### 1-4. 성공 기준 (Success Criteria)

- 회귀 모델: RMSE 최소화, R² ≥ 0.5 이상
- 분류 모델: F1-score ≥ 0.70 (클래스 불균형 고려)
- 통계 검증: 카테고리 간 트렌딩 지속 시간 차이의 통계적 유의성 확인 (Kruskal-Wallis p < 0.05)

### STEP 1. 무결성 확인 및 한국 크리에이터 필터

#### 시간대 변환 (UTC → KST)

원본 데이터가 영국 표준시(UTC) 기준으로 수집되어 한국 시간(KST, UTC+9)으로 변환하였습니다.  
업로드 시간대 피처(hour_sin/cos, published_weekday)의 정확성을 위해 필수적입니다.

```python
df['collection_date'] = (
    pd.to_datetime(df['collection_date'], utc=True)
    .dt.tz_convert('Asia/Seoul')
    .dt.tz_localize(None)
)
df['published_at'] = (
    pd.to_datetime(df['published_at'], utc=True)
    .dt.tz_convert('Asia/Seoul')
    .dt.tz_localize(None)
)
```

#### 한국 크리에이터 필터

분석 대상을 '한국 광고주가 집행 가능한 한국 크리에이터 영상'으로 한정하기 위해 3가지 조건을 OR 조합하여 필터링합니다.

```python
korean_title_mask   = df['title'].str.contains(r'[가-힣]', na=False)
korean_channel_mask = df['channel_title'].str.contains(r'[가-힣]', na=False)
ko_lang_mask        = df['default_language'] == 'ko'
df = df[ko_lang_mask | korean_title_mask | korean_channel_mask]
```

| 조건 | 변수 | 설명 |
|------|------|------|
| 한국어 제목 | korean_title_mask | 제목에 한글(가-힣) 포함 여부 |
| 한국어 채널명 | korean_channel_mask | 채널명에 한글(가-힣) 포함 여부 |
| 언어 설정 ko | ko_lang_mask | default_language == 'ko' |

> OR 조합을 사용한 이유: 한국 크리에이터가 영어 제목을 사용하더라도 채널명이 한국어인 경우가 많으며, 언어 설정을 하지 않은 영상(결측 63%)도 실제로는 한국 콘텐츠인 경우가 대부분이기 때문입니다.

### STEP 3. 재진입 이벤트 분리 (6시간 기준)

같은 영상이 트렌딩에 올랐다가 이탈한 후 재진입하는 경우를 별도 이벤트로 분리합니다.

#### 6시간 기준의 근거

수집 주기가 6시간(하루 4회)이므로, 연속된 스냅샷 간 간격이 6시간을 초과한다는 것은 중간 수집 시점에 트렌딩 목록에 없었다는 의미입니다. 즉 **6시간은 임의적 기준이 아니라 수집 주기 자체가 기준**입니다.

```python
df = df.sort_values(['video_id', 'collection_date'])

df['time_gap_h'] = (
    df.groupby('video_id')['collection_date']
      .diff()
      .dt.total_seconds() / 3600
)

# 6시간 초과 공백 → 새 이벤트 시작
df['new_event'] = (df['time_gap_h'] > 6) | (df['time_gap_h'].isna())
df['event_id']  = df.groupby('video_id')['new_event'].cumsum().astype(int)
```

### STEP 4. view_growth_24h 추출 (event_id 기준)

이벤트 집계 이전에 각 이벤트의 트렌딩 진입 후 24시간 이내 조회수 증가량을 추출합니다.  
이벤트 집계 후에는 정확한 계산이 불가능하므로 **원본 df 기준으로 먼저 계산**합니다.

```python
# 이벤트별 T0 시각 계산
T0_time = (
    df.groupby(['video_id', 'event_id'])['collection_date']
    .min().rename('T0_time').reset_index()
)
df = df.merge(T0_time, on=['video_id', 'event_id'], how='left')

# T0 시점 조회수 (첫 관측값)
T0_view = (
    df.sort_values(['video_id', 'event_id', 'collection_date'])
    .groupby(['video_id', 'event_id'])['view_count']
    .first().rename('T0_view').reset_index()
)

# T0+24h 이내 마지막 조회수
df_24h = df[df['collection_date'] <= df['T0_time'] + pd.Timedelta(hours=24)]
view_at_24h = (
    df_24h.groupby(['video_id', 'event_id'])['view_count']
    .last().rename('view_at_24h').reset_index()
)

view_growth = T0_view.merge(view_at_24h, on=['video_id', 'event_id'], how='left')
view_growth['view_growth_24h'] = (
    view_growth['view_at_24h'] - view_growth['T0_view']
).clip(lower=0)
```

### STEP 5. 이벤트 집계

스냅샷 단위 데이터(820,630행)를 `video_id × event_id` 단위로 집계하여 이벤트 테이블을 생성합니다.

```python
events = df.groupby(['video_id', 'event_id']).agg(
    T0            = ('collection_date', 'min'),
    T_end         = ('collection_date', 'max'),
    channel_id    = ('channel_id', 'first'),
    channel_title = ('channel_title', 'first'),
    category_id   = ('category_id', 'first'),
    category_group= ('category_group', 'first'),
    title         = ('title', 'first'),
    published_at  = ('published_at', 'first'),
    entry_rank    = ('rank', 'first'),       # 진입 당시 순위
    best_rank     = ('rank', 'min'),         # 최고 순위 (숫자 작을수록 좋음)
    peak_view     = ('view_count', 'max'),   # ⚠️ 분석용만 (예측 피처 제외)
    T0_view       = ('view_count', 'first'), # 진입 당시 조회수
    T0_comment    = ('comment_count', 'first'), # 진입 당시 댓글 수
    T_end_view    = ('view_count', 'last'),  # 종료 당시 조회수
    T_end_comment = ('comment_count', 'last'),
).reset_index()

# trending_duration_h: 실제 시간차 계산 (날짜 단위 추정 방식 X)
events['trending_duration_h'] = (
    events['T_end'] - events['T0']
).dt.total_seconds() / 3600

# 최소 6h 보정 (당일 진입/종료 케이스)
events['trending_duration_h'] = events['trending_duration_h'].clip(lower=6)
```

> ⚠️ `peak_view`는 트렌딩이 끝나야 알 수 있는 미래 정보이므로 모델 입력 피처에서 제외

### STEP 6. 파생 피처 생성 및 Winsorization

#### 생성된 파생 피처

| 피처명 | 계산식 | 의미 | 비고 |
|--------|--------|------|------|
| latency_to_trend_h | (T0 - published_at).seconds / 3600 | 업로드 후 트렌딩 진입까지 걸린 시간 | |
| view_velocity | (T_end_view - T0_view) / trending_duration_h | 시간당 조회수 증가율 | peak_view 대신 T_end_view 사용 (왜곡 방지) |
| engagement_ratio | T0_comment / T0_view | 진입 당시 조회수 대비 댓글 비율 | T_end_comment 사용 시 target leakage 발생 |
| published_weekday | published_at.dt.dayofweek | 업로드 요일 (0=월 ~ 6=일) | |
| hour_sin | sin(2π × hour / 24) | 업로드 시간 사이클 인코딩 | 순환 구조 표현 |
| hour_cos | cos(2π × hour / 24) | 업로드 시간 사이클 인코딩 | 순환 구조 표현 |
| has_korean_title | 한글(가-힣) 포함 여부 → 0/1 | 한국어 제목 여부 | 영어 제목 크리에이터 구분 |
| entry_rank_log | log(1 + entry_rank) | 진입 순위 로그 변환 | 왜도 완화 |
| view_growth_24h | view_at_24h - T0_view | 초기 24시간 조회수 증가량 | STEP 4에서 계산 후 JOIN |

> `engagement_ratio`에 `T0_comment`를 사용한 이유: `T_end_comment`는 트렌딩 종료 후에야 알 수 있는 값으로, 이를 입력 피처로 사용하면 target leakage가 발생합니다.

> `hour_sin/cos`를 사용한 이유: 시간(0~23)을 그대로 수치로 쓰면 모델이 "23시와 0시가 멀다"고 학습하지만, sin/cos 변환으로 원형 시간 구조를 표현하면 "23시와 0시는 가깝다"는 순환 구조를 올바르게 학습할 수 있습니다.

#### Winsorization 적용

YouTube 데이터 특성상 조회수, 성장 속도 등이 오른쪽 꼬리가 매우 긴 분포(right-skewed)를 보입니다. 이상치가 모델 학습에 미치는 영향을 줄이기 위해 Winsorization을 적용하되, **원본값은 분석용으로 보존**합니다.

**5%/95% 기준 선택 근거**

| 기준 | 특징 | 선택 여부 |
|------|------|----------|
| 1%/99% | 너무 관대 → 이상치 영향 잔존 | ✗ |
| 5%/95% | 적당한 수준 → ML 전처리 표준, 이상치 제거와 데이터 보존의 균형 | ✅ |
| 10%/90% | 너무 공격적 → 정상 데이터까지 손실 | ✗ |

**적용 변수**

| 원본 변수명 | Winsorized 변수명 | 적용 근거 |
|------------|-----------------|----------|
| trending_duration_h | trending_duration_h_wins | 롱런 영상 이상치 영향 완화 |
| view_velocity | view_velocity_wins | 바이럴 영상 폭발적 증가 완화 |
| engagement_ratio | engagement_ratio_wins | 소규모 영상 댓글 비율 이상치 완화 |
| T_end_view | view_count_wins | 조회수 규모 차이 완화 |
| view_growth_24h | view_growth_24h_wins | 초기 바이럴 이상치 완화 |

### STEP 7. TDI 산출 및 tdi_label 생성

#### TDI (Trending Duration Index) 정의

단순 지속 시간이 아니라 **카테고리 난이도와 트렌딩 순위를 동시에 반영한 트렌딩 영향력 지수**입니다.

```
TDI = clip(trending_duration_h / cat_q95_duration, 0, 1)
      × (1 - (best_rank - 1) / 200)

지속성 점수: 같은 카테고리 내에서 얼마나 오래 버텼는가 (0~1)
순위 점수:   1위=1.0, 100위=0.505, 200위=0.0
```

```python
cat_q95 = events.groupby('category_group')['trending_duration_h'].quantile(0.95)
events['cat_q95_duration'] = events['category_group'].map(cat_q95)

events['TDI'] = (
    (events['trending_duration_h'] / events['cat_q95_duration']).clip(0, 1)
    * (1 - (events['best_rank'] - 1) / 200)
).clip(0, 1)
```

**cat_q95를 분모로 선택한 이유**  
`cat_max`(최대값)를 분모로 사용하면 이상치 영상 1개로 인해 전체 카테고리의 TDI가 0에 수렴하는 문제가 발생합니다. 95% quantile을 사용하면 상위 5% 이상치의 영향을 차단하면서도 카테고리의 실질적 상한을 반영할 수 있습니다.

### STEP 8. saturation_index 계산

날짜별 카테고리 내 트렌딩 영상 수를 기반으로 카테고리 경쟁 밀도를 수치화합니다.

```python
events['T0_date'] = events['T0'].dt.date

cat_sat = (events.groupby(['T0_date', 'category_group'])['video_id']
                 .nunique()
                 .reset_index(name='video_count'))

cat_sat['saturation_index'] = (
    cat_sat.groupby('category_group')['video_count']
           .transform(lambda x: x / x.rolling(30, min_periods=1).max())
)

events = events.merge(
    cat_sat[['T0_date', 'category_group', 'saturation_index']],
    on=['T0_date', 'category_group'], how='left'
)
```
